# 📱 Извлечение подписей из оригинального APK APatch

## 🎯 Цель

Теперь у нас есть **оригинальный APK** из официального GitHub Actions billd, мы можем извлечь **ТОЧНЫЕ** подписи и решить проблему зависания раз и навсегда!

## 📥 Шаг 1: Скачивание оригинального APK

### Из GitHub Actions:
1. Перейдите: https://github.com/bmax121/APatch/actions/runs/16396403844
2. Скачайте артефакт с APK файлом
3. Извлеките APK из архива

### Или из Releases:
1. Перейдите: https://github.com/bmax121/APatch/releases
2. Скачайте последний релиз APK
3. Сохраните как `APatch-original.apk`

## ⚡ Шаг 2: Быстрое извлечение (БЕЗ зависимостей)

Используйте быстрый скрипт **без** внешних зависимостей:

```bash
# Скачайте скрипт
wget https://raw.githubusercontent.com/ваш-пользователь/ваш-репо/main/quick_extract_apk_signatures.py

# Сделайте исполняемым
chmod +x quick_extract_apk_signatures.py

# Запустите анализ
python3 quick_extract_apk_signatures.py APatch-original.apk
```

**Результат:**
- ✅ Извлечет все подписи без установки дополнительных инструментов
- ✅ Покажет готовые значения для GitHub Secrets
- ✅ Сохранит результаты в файл

## 🔧 Шаг 3: Полный анализ (с инструментами)

Для более детального анализа используйте полный скрипт:

```bash
# Установите зависимости (Ubuntu/Debian)
sudo apt-get update
sudo apt-get install aapt openssl

# Запустите полный анализ
python3 extract_signatures_from_apk.py APatch-original.apk
```

**Дополнительные возможности:**
- ✅ Анализ сертификатов через OpenSSL
- ✅ Извлечение отпечатков пальцев
- ✅ Поиск встроенных подписей в коде
- ✅ Сохранение в JSON формате

## 📋 Шаг 4: Копирование секретов

После извлечения скопируйте найденные значения в GitHub Secrets:

### 🔗 Ваш репозиторий: 
**https://github.com/AngelOfLlife/APatch/settings/secrets/actions**

### 🔑 Обновите эти секреты:

```bash
# Замените существующие подписи на извлеченные из оригинального APK
APATCH_REAL_SIGNATURE=<извлеченная_подпись_1>
APATCH_CODE_SIGNATURE=<извлеченная_подпись_2>

# Остальные секреты остаются прежними
SIGNING_KEY=<ваш_keystore_base64>
ALIAS=apatch
KEY_STORE_PASSWORD=apatch123
KEY_PASSWORD=apatch123
APATCH_PACKAGE_NAME=me.bmax.apatch
APATCH_APP_CLASS=APApplication
APATCH_SIGNATURE_METHOD=verifyAppSignature
```

## 🎯 Шаг 5: Тестирование

После обновления секретов:

1. **Запустите любой workflow** (например, `ultimate_fix_apatch_build.yml`)
2. **Проверьте логи** - должны исчезнуть ошибки патчинга подписи
3. **Скачайте APK** из артефактов
4. **Установите и тестируйте** - зависание должно исчезнуть

## 🔍 Что ищет скрипт?

### ⚡ Quick Extract ищет:
- **SHA1/SHA256 хеши** всего APK файла
- **Подписи из META-INF** директории
- **Встроенные строки** в .dex файлах
- **Известные паттерны** APatch
- **Base64 кандидаты** на подписи

### 🔧 Full Extract дополнительно:
- **Отпечатки сертификатов** через OpenSSL
- **PEM сертификаты** и их хеши
- **Детальный анализ** манифеста
- **Хеши без подписи** (unsigned)
- **Бинарный поиск** паттернов

1. **Используйте разные варианты** - если первая подпись не работает, попробуйте следующую
2. **Проверяйте логи** workflow - они покажут, какие подписи заменяются
3. **Тестируйте поэтапно** - сначала обновите подписи, потом запустите сборку
4. **Сохраняйте оригиналы** - держите копию рабочих секретов на случай отката
